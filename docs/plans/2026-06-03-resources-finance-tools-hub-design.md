# Resources Finance Tools Hub Design

## Purpose

Turn `/resources/` from a mixed internal library into a public hub for people looking for practical finance automation tools. The page should help AP controllers, finance operators, and decision makers find the right open resource quickly, while strengthening the internal link cluster around finance automation.

## Search Target

Ahrefs data points to the page targeting a hub intent rather than the generic word "resources".

- Primary keyword: `finance automation tools`
- Secondary keywords: `invoice automation tools`, `accounts payable tools`, `accounts payable automation tools`, `accounts payable controls`
- Supporting clusters: `invoice processing automation`, `invoice OCR`, `invoice parser`, `compliance automation`

The page should not try to rank for broad "resources" intent.

## Page Direction

Use the title and H1 to make the page immediately useful:

- Title: `Free Finance Automation Tools for AP Controls and Invoice Workflows`
- H1: `Free finance automation tools for AP teams`

The page should present available resources only, including the live three-way match tool. Remove public placeholder language, coming-soon cards, and internal phrases such as "pillar", "RegOps will grow", "proof notes", or "resource library" as the main promise.

## Structure

1. Hero: direct promise around free finance automation tools for invoice data, AP controls, invoice processing, and close-cost analysis.
2. Featured path: a compact "Start with the problem" section that maps user intent to the correct resource.
3. Live resources: show only available tools and guides, grouped by use case rather than content taxonomy.
4. Compliance proof: keep the EUDR case note as a secondary compliance automation example, not equal weight with finance.
5. Final CTA: soft workflow CTA plus a link back to the finance automation pillar page.

## Design

Keep the existing brand language: cream/white surfaces, green system, restrained amber accents, serif headings, compact editorial spacing. The page should feel like a practical index, not a landing page clone.

Use dense link panels, problem-based rows, and compact resource cards. Avoid decorative placeholders and blank grid gaps.

## Content Rules

- No coming-soon resources.
- No fake metrics.
- No internal strategy language.
- No unsupported Xero-agent claims.
- Use natural anchors to `/finance-automation/`, `/resources/invoice-extractor/`, `/resources/accounts-payable-controls/`, `/glossary/three-way-match/`, `/resources/invoice-processing-automation/`, `/resources/cost-to-close-calculator/`, and `/work/eudr-compliance-bridge/`.

## Verification

- Run lint.
- Run build.
- Confirm `/resources/` renders.
- Confirm no visible `coming soon` items appear.
- Confirm old internal wording is gone from the page.
- Confirm all visible links resolve to existing internal routes.
